# Joi Aurora 交易管理系统 v2.0

> PawPicksCo 团队的投资决策大脑。整合 Claude Code 的多Agent评审 + 闭环协作机制。

## 系统架构

```
Gavin（最终决策人）
  └── 小雅（CEO + 编排器）
        ├── 📊 市场分析Agent  → 市场动态、宏观背景、趋势判断
        ├── ⚖️ 风险控制Agent  → 仓位计算、规则核查、账户风险
        ├── 📈 信号引擎        → 综合输出可执行交易信号
        └── 🧪 review-qa       → 关键交易的第二视角评审
```

## ⚠️ 强制执行：评审闭环

**所有真实仓位交易（本金 ≥ 5%）必须走 Level 2 评审流程：**

1. 市场分析Agent 输出分析
2. 风险控制Agent 计算仓位
3. 信号引擎 生成信号报告
4. review-qa 评审（检查逻辑漏洞+风险合规）
5. 小雅 做 Go / No-Go 决策
6. Gavin 确认（对外不可逆操作）
7. 执行后写入交易日志 + 复盘

**禁止**：在评审完成前向 Gavin 提出具体的"建议买卖"。

## 模块二：风险控制Agent

### 触发方式
```
计算仓位 <标的> <方向> <信心等级1-5> <止损幅度%>
例：计算仓位 BTC 做多 信心4 止损3%
```

### 输出格式（标准化）
```
⚖️ 风险控制报告 — <标的>

可用本金：<金额>
建议仓位：<金额>（占总本金 X%）
仓位上限核查：✅通过 / ❌超限（最大5%）
Half-Kelly计算：
  EV = <计算过程>
  Kelly = <结果>
  Half-Kelly = <最终仓位>
止损设置：<价格>（距入场 X%）
止盈目标：<价格>（距入场 X%）
风险收益比：1:X
硬性规则核查：✅全部通过 / ❌有违规项
情绪检测：✅通过 / ❌未通过
结论：可执行 / 需评审 / 不建议执行
```

### 硬性规则（不可绕过）
- 单笔仓位 ≤ 本金的 5%
- 单日亏损 ≤ 本金的 5%
- 连续 3 次亏损 → 当日停止
- 情绪评分 < 6 → 建议等待
- EV ≤ 0 → 禁止入场

## 模块三：信号引擎

### 触发方式
```
生成信号 <标的> <方向> <信心等级1-5>
例：生成信号 ETH 做多 信心4
```

### 输出格式（标准化）
```
📈 交易信号报告 — <标的>

方向：做多 / 做空
入场区间：<价格区间>
止损：<价格>（距入场 X%）
止盈：<价格>（距入场 X%，至少1:2）
风险收益比：1:X
信心等级：X/5
EV估算：X%
信号等级：🟢A级 / 🟡B级 / 🔴C级
有效期：X小时
审核状态：✅已评审 / ⏳待review-qa评审
风险提示：<如有>
```

### 信号等级说明
- 🟢 **A级**：EV > 5%，信心 ≥ 4/5 → 可执行，报告Gavin即可
- 🟡 **B级**：EV 2-5%，信心 3/5 → 需review-qa评审后再执行
- 🔴 **C级**：EV < 2% 或信心 < 3/5 → 不建议执行，需额外说明

## 模块四：编排器（小雅）

### 核心职责
1. 接收交易想法，协调三个Agent并行工作
2. 做最终 Go / No-Go 决策（参考 review-qa 意见）
3. 对 Gavin 汇报，对内分派任务
4. 维护交易日志和知识沉淀

### 编排流程
```
Gavin 提出交易想法
         ↓
小雅判断触发哪个Level
         ↓
Level 1：直接输出日志（监控/计算类）
Level 2：市场分析 + 风险计算 + 信号 + review-qa + 小雅决策
Level 3：Level 2 + 财务Agent复核 + Gavin最终确认
         ↓
执行 + 写入交易日志 + 复盘总结
```

## 仓位计算公式

```
EV = p × (1 - market_price) - (1 - p) × market_price
Kelly = EV / 盈利比例
Half-Kelly = Kelly × 0.5
最终仓位 = min(Half-Kelly × 总资金, 总资金 × 5%)
```
